# Quick Start

Get RTK up and running in under 5 minutes. This guide walks you through installation, verification, initialization, and your first token-optimized commands.

## Prerequisites

- macOS, Linux, or Windows
- Command-line access
- Claude Code, Cursor, or another AI coding assistant (optional but recommended)

## Step 1: Install RTK

Choose your preferred installation method:

<CodeGroup>
```bash Homebrew (macOS/Linux)
brew install rtk
```

```bash Quick Install (Linux/macOS)
curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/master/install.sh | sh
```

```bash Cargo (from source)
cargo install --git https://github.com/rtk-ai/rtk
```
</CodeGroup>

<Note>
If you already have RTK installed, verify it's the correct package:

```bash
rtk gain
```

If this command works (even if it says "No tracking data yet"), you're good to go. Skip to [Step 3](#step-3-initialize-rtk).
</Note>

## Step 2: Verify Installation

Confirm RTK is installed correctly:

```bash
rtk --version
```

<CodeGroup>
```bash Expected Output
rtk 0.24.0
```

```bash Verify It's Token Killer (Not Type Kit)
rtk gain
```

```text Expected Output
No tracking data yet.
Run some rtk commands to start tracking savings.
```
</CodeGroup>

<Warning>
If `rtk gain` returns "command not found" but `rtk --version` works, you installed the **wrong** RTK (Rust Type Kit).

Uninstall and reinstall the correct package:
```bash
cargo uninstall rtk
cargo install --git https://github.com/rtk-ai/rtk
```
</Warning>

## Step 3: Initialize RTK

Set up RTK to work automatically with your AI coding assistant:

```bash
rtk init --global
```

<Steps>
  <Step title="Run the init command">
    ```bash
    rtk init --global
    ```
    
    This installs a hook that automatically rewrites commands across all projects.
  </Step>
  
  <Step title="Answer the prompt">
    ```
    Patch settings.json? [y/N]: y
    ```
    
    Type `y` and press Enter to register the hook with Claude Code.
  </Step>
  
  <Step title="Restart Claude Code">
    Close and reopen Claude Code for the hook to take effect.
  </Step>
</Steps>

<Note>
**What just happened?**

- **Hook installed:** `~/.claude/hooks/rtk-rewrite.sh` now intercepts bash commands
- **Context optimized:** `~/.claude/RTK.md` added (only 10 lines, saves ~2000 tokens vs old method)
- **Settings patched:** `~/.claude/settings.json` now registers the hook
- **Backup created:** Original settings saved to `~/.claude/settings.json.bak`
</Note>

## Tips for Maximum Savings

<Steps>
  <Step title="Use hooks for automatic optimization">
    With `rtk init --global`, you never need to type `rtk` manually - all commands are rewritten automatically.
  </Step>
  
  <Step title="Prefix commands manually if hooks aren't installed">
    If you skipped the hook setup, manually prefix commands:
    ```bash
    rtk git status
    rtk npm test
    rtk docker ps
    ```
  </Step>
  
  <Step title="Check savings regularly">
    Run `rtk gain` weekly to see your cumulative savings and identify high-impact commands.
  </Step>
  
  <Step title="Report missing commands">
    If you find a command RTK doesn't optimize, [open an issue](https://github.com/rtk-ai/rtk/issues) - we're always adding more!
  </Step>
</Steps>

## Common Workflows

### Git Workflow

```bash
# Check status
rtk git status

# View recent commits
rtk git log -10

# See changes
rtk git diff

# Stage and commit
rtk git add .
rtk git commit -m "Your message"

# Push to remote
rtk git push
```

### Test-Driven Development

```bash
# Run tests (show failures only)
rtk test cargo test
rtk test npm test
rtk pytest
rtk go test

# Run linters
rtk lint
rtk cargo clippy
rtk ruff check
```

### Package Management

```bash
# List dependencies
rtk pnpm list
rtk pip list

# Check for updates
rtk pnpm outdated
rtk pip outdated

# Install packages (quiet mode)
rtk pnpm install react
rtk pip install requests
```

### Container Operations

```bash
# List containers
rtk docker ps

# View logs (deduplicated)
rtk docker logs my-container

# Kubernetes
rtk kubectl get pods
rtk kubectl logs my-pod
```
