# RTK - Rust Token Killer

RTK is a high-performance CLI tool written in Rust that dramatically reduces LLM token consumption by filtering and compressing command outputs before they reach your AI coding assistant's context.

<Note>
**Token Savings:** RTK achieves **60-90% reduction** in tokens on common development operations.

A typical 30-minute Claude Code session goes from **~150,000 tokens** down to **~45,000 tokens** - a **70% reduction**.
</Note>

## Why RTK Matters

AI coding assistants like Claude Code, Cursor, and Copilot consume massive amounts of tokens when reading command outputs. Every `git status`, `npm test`, or `ls -la` sends verbose output into the LLM context - wasting tokens, hitting rate limits, and slowing down your workflow.

RTK sits between your AI assistant and the shell, intelligently filtering outputs to preserve only what matters:

```
Without RTK:
  ┌──────────┐  git status     ┌──────────┐  git status  ┌──────────┐
  │  Claude  │ ─────────────── │  shell   │ ──────────── │   git    │
  │   LLM    │                 │          │              │  (CLI)   │
  └──────────┘                 └──────────┘              └──────────┘
        ▲                                                      │
        │              ~2,000 tokens (raw output)              │
        └──────────────────────────────────────────────────────┘

With RTK:
  ┌──────────┐  git status     ┌──────────┐  git status  ┌──────────┐
  │  Claude  │ ─────────────── │   RTK    │ ──────────── │   git    │
  │   LLM    │                 │  (proxy) │              │  (CLI)   │
  └──────────┘                 └──────────┘              └──────────┘
        ▲                           │  ~2,000 tokens raw       │
        │                           └──────────────────────────┘
        │  ~200 tokens (filtered)   filter · group · dedup · truncate
        └───────────────────────────────────────────────────────
```

## How It Works

RTK applies four intelligent strategies based on command type:

<Steps>
  <Step title="Smart Filtering">
    Removes noise like comments, whitespace, boilerplate progress bars, and redundant metadata.
  </Step>
  
  <Step title="Grouping">
    Aggregates similar items - files by directory, errors by type, commits by author.
  </Step>
  
  <Step title="Truncation">
    Keeps relevant context, cuts redundancy. Show first/last 10 lines instead of 1000.
  </Step>
  
  <Step title="Deduplication">
    Collapses repeated log lines with counts: `Error: timeout (×347)` instead of 347 lines.
  </Step>
</Steps>

### Real-World Example


**RTK git push output (1 line, ~10 tokens - 95% reduction):**
```bash
$ rtk git push

```
