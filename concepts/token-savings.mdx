## Token Estimation Formula

RTK uses a simple heuristic for token estimation:

```rust
fn estimate_tokens(text: &str) -> usize {
    (text.len() as f64 / 4.0).ceil() as usize
}
```

**Approximation:** ~4 characters per token (GPT-style tokenization)

This is a rough estimate but accurate enough for tracking purposes. Real tokenization varies by model (Claude uses a different tokenizer than GPT), but the 4:1 ratio is a good average.

## Best Practices for Maximum Savings

<Steps>
  <Step title="Use the hook">
    Install `rtk init -g` for 100% adoption. Manual prefixing leads to ~60-70% adoption.
  </Step>
  
  <Step title="Prefer rtk for repetitive commands">
    Commands you run 10+ times per session (git status, test, lint) benefit most.
  </Step>
  
  <Step title="Use aggressive filter for code reading">
    `rtk read file.rs -l aggressive` strips function bodies, saving 60-90% on large files.
  </Step>
  
  <Step title="Let failures surface naturally">
    Don't use `-v` flags unless debugging. Compact output is optimized for LLMs.
  </Step>
  
  <Step title="Review savings monthly">
    Run `rtk gain --monthly` to track trends and identify optimization opportunities.
  </Step>
</Steps>
